# PicFolderBot

Telegram-бот на Go для загрузки изображений дизайнеров в структурированные папки Яндекс.Диска.

## Что умеет MVP

- Принимает фото + подпись и пытается извлечь `товар`, `цвет`, `папка`.
- Если параметров не хватает, предлагает выбор через inline-кнопки.
- Поддерживает пошаговый режим через `/upload`.
- Позволяет создавать папки на каждом уровне через кнопку `+Добавить папку`.
- Поддерживает кнопку `Назад` на каждом шаге и пагинацию длинных списков.
- Загружает изображение в путь:
  `<YANDEX_ROOT_PATH>/<Товар>/<Цвет>/<Раздел>/filename.jpg`
  (бот использует только реально существующие папки и не создает разделы автоматически при загрузке)

## Переменные окружения

Скопируйте `.env.example` в `.env` и заполните:

- `TELEGRAM_BOT_TOKEN` — токен бота из BotFather.
- `YANDEX_OAUTH_TOKEN` — OAuth-токен Яндекс.Диска.
- `YANDEX_ROOT_PATH` — корневая папка, например `disk:/Товары InnoGoods`.
- `YANDEX_TIMEOUT_SEC` — timeout запросов к API Яндекса.

## Как получить YANDEX_OAUTH_TOKEN

1. Создайте OAuth-приложение в Яндекс OAuth (тип "Веб-сервис").
2. Запросите scope для Яндекс.Диска: `cloud_api:disk.read cloud_api:disk.write`.
3. Получите access token через OAuth flow Яндекса.
4. Запишите токен в `.env` как `YANDEX_OAUTH_TOKEN`.

Если токен или диск нужно переключить на другой аккаунт, достаточно поменять
`YANDEX_OAUTH_TOKEN` и/или `YANDEX_ROOT_PATH` в `.env` и перезапустить контейнер.

## Локальный запуск

```bash
go mod tidy
go run .
```

## Docker

```bash
docker compose up --build -d
docker compose logs -f
```

## Деплой на сервер (кратко)

1. Скопировать проект на сервер.
2. Создать `.env`.
3. Выполнить:
   - `docker compose build`
   - `docker compose up -d`
4. Проверить логи: `docker compose logs -f`.

## Регрессионный чеклист перед релизом

Проверьте минимум эти сценарии:

1. `/upload` без выбранного пути: выбор `товар -> цвет -> раздел`, загрузка 1 файла.
2. Пакетная загрузка (media group):
   - сначала путь, потом пакет;
   - сначала пакет, потом путь (deferred upload).
3. Разделы типа "Титульники":
   - перед загрузкой появляется шаг переименования;
   - "Без изменений" работает;
   - новое имя заменяет исходное имя файла.
4. Кнопка `Назад`:
   - в списках, в экране успеха, в rename-экране;
   - не теряется контекст выбранного пути.
5. Кнопка `🔄 Обновить список`:
   - после создания новой папки она появляется сразу на соответствующем уровне.
6. `/search`:
   - точный матч (мгновенный переход),
   - неоднозначный ввод (подсказки),
   - полный путь из нескольких частей.
7. `🕘 Последние`:
   - путь записывается после успешной загрузки,
   - выбор из недавних корректно восстанавливает путь.
8. Ошибки:
   - нет "сырых" технических ошибок для пользователя;
   - при сетевых сбоях есть понятный текст и повторяемость действий.
